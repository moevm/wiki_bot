Рейтинговая система по дисциплине “Информатика” осень 2022

Перед началом выполнения практических заданий по дисциплине “Информатика” в первом семестре студент должен ознакомиться с данными правилами и пройти тест в первом модуле основного курса.

Студенты, не прошедшие тест, считаются неприступившими к выполнению практической части  дисциплины.

Время везде в данном документе указано в часовом поясе GMT+3

Оценка по дисциплине формируется по результатам работы студента в течение семестра на рейтинговой основе и складывается из выполнения следующих компонент (этапов):

  • Выполнение лабораторных работ.
  • Защита лабораторных работ.
  • Написание контрольных работ.
  • Прохождение основного онлайн-курса на e.moevm.info.
  • Обязательное прохождение онлайн-курса по Git.
  • Прохождение онлайн-курса по Linux.

За выполнение каждой компоненты (этапа) студент получает определенное количество баллов, сумма которых определяет оценку за дифференцированный зачет.

На лабораторных работах студент либо защищает лабораторные работы, либо проходит задачи онлайн-курса.

Раздел “Форматирование отчета” также является обязательным к прохождению (как и курс по Git) для допуска к защите лабораторной работы №1.

Важно: чтобы за прохождение онлайн-курса были зачтены все баллы студента, студент должен строго до дедлайна убедиться, что все открытые попытки в онлайн-курсе были закрыты. К сожалению, баллы за открытые попытки технически невозможно учесть.

Как производится подсчет оценки за курс. Для каждого раздела определяется количество решенных задач в соответствии с дедлайнами (например, количество решенных задач первого раздела будет зафиксировано 24 октября в 00:00). Далее количество решенных задач суммируется и вычисляется балл по формуле: 

=ROUNDDOWN(Количество решенных задач по всем разделам курса в соответствии с дедлайнами / Количество задач всего в курсе * 5)

Для получения допуска к защите лабораторной работы студент должен создать PR с кодом, который был принят системой проверки (стоит метка Passed), и отчетом (см. табл. 1, пункт 1) в срок до дедлайна (см. табл. 2). В таком случае студент защищает лабораторную работу на следующем по расписанию занятии, после даты создания PR.

  • 0 баллов за защиту лабораторной работы можно пересдать на следующих занятиях по расписанию до дедлайна.
  • баллы 1 и 2 не подлежат перезащите.
  • Защита лабораторной работы после дедлайна возможна однократно только для тех, у кого не было ни одной попытки до этого момента.
  • Если обнаруживается, что студент защищал лабораторную, не решив задачу в дистанционном курсе, то результат защиты аннулируется без возможности перезащиты.
  • Если обнаруживается, что студент выполнил лабораторную работу или задания на защиту лабораторной работы не самостоятельно, то результат защиты аннулируется без возможности перезащиты.

Примечание: пулл реквесты с метками Failed и Violated не проверяются преподавателем и соответствующие ЛР не допускаются к защите. Чтобы преподаватель проверил пулл реквест, нужно исправить все замечания проверяющей системы, добиться метки Passed.

Важно: на дистанционных занятиях (лекциях, лабораторных, консультациях и пр.), где бы они не проходили (pruffme, zoom, discord и т.п.) студент должен указывать свои настоящие имя и фамилию, номер группы. В противном случае преподаватель не сможет идентифицировать студента и, соответственно, студент не будет допущен до занятий.

Приглашение в репозиторий github.

Рассылка приглашений в репозитории состоялась 13 сентября 2022. Будьте внимательны: приглашение в репозиторий истекает через 1 неделю (7 дней). Если вы не приняли приглашение в репозиторий на github, и оно истекло, преподаватель вышлет повторное приглашение в срок до 2 недель с момента получения вашего письма на почту с просьбой отправить приглашение повторно (письмо должно быть с указанной темой и соблюдением правил составления писем http://se.moevm.info/doku.php/inf:communication_rules).

 

Дедлайн создания всех PR (с целью получения 2 баллов за них), исправления замечаний к PR, а также прохождения основного и других онлайн курсов:

25 декабря 23:59:59

Если после этого срока в пулл реквесте останутся замечания, баллы за него уже нельзя будет получить.

Примечание: баллы за лабораторную работу будут зафиксированы только после того, как преподаватель смержит пулл реквест. Пока пулл реквест не смержен, баллов у студента нет, даже если он защищал лабораторную работу.

Примеры расчета оценки:

  • Удовлетворительно:

Подготовлены PRs ко всем лабораторным работам (2*3 = 6 баллов) + Пройдена часть основного онлайн-курса курса (3 балла) + Написана контрольная работа на на 2 балла + Пройден частично онлайн-курс по Git (2 балла) + Пройден частично онлайн-курс по Linux (2 балла) = 15 баллов

или

Пройден полностью онлайн-курс по Git (3 балла) + Пройден полностью онлайн-курс по Linux (3 балла) + Пройден полностью основной онлайн-курс (5 баллов) + сделан один PR (2 балла) = 13 баллов

  • Хорошо:

Пройден полностью онлайн-курс по Git (3 балла) + Пройден полностью основной онлайн-курс (5 баллов) + Написана контрольная работа на 4 балла + Подготовлены PRs ко всем лабораторным работам (6 баллов) + Защищены две лабораторные по 2 балла (4 балла) = 22 балла

  • Отлично:

Пройден полностью основной онлайн-курс (5 баллов) + Написана контрольная работа на 5 баллов + Пройден полностью онлайн-курс по Git (3 балла) + Пройден полностью онлайн-курс по Linux (3 балла) + Подготовлены PRs ко всем лабораторным работам (6 баллов) + Защищены три лабораторные по 2 балла (6 баллов) =  28 баллов

Правила оценивания лабораторных

Полная инструкция о дистанционной защите лабораторной работы.

Защита лабораторных состоит из:

  • Корректного pull-request’а (исходный код соответствует исходному коду принятому системой проверки онлайна-курса + корректно оформленный отчет в формате .pdf)
  • Выполненных заданий на защиту
  • Ответа на устные вопросов без подготовки (вопросы связаны с теорией, заданиями на защиту или программой, написанной в рамках лабораторной работы)
