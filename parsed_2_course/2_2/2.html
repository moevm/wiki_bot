Составляющие оценки

Оценка по дисциплине формируется по результатам работы студента в течение семестра на рейтинговой основе и складывается из выполнения следующих компонент (этапов):

  • Посещение консультаций (лекций)
  • Защита лабораторных работ
  • Написание контрольной работы
  • Прохождение онлайн курса на moodle  http://e.moevm.info/course/view.php?id=45

За выполнение каждой компоненты (этапа) студент получает определенное количество баллов, сумма которых определяет оценку за экзамен. Первичные баллы используются для оценивания лабораторных и контрольной работ, тестовые - для оценивания онлайн-курсов и лекционных занятий. Перевод из первичных баллов в тестовые производится с коэффициентом 3.

На лабораторных занятиях студент либо защищает лабораторные работы, либо проходит задачи онлайн-курса.

Изменение правил дедлайнов и защит лабораторных работ (начиная с л/р №2)

Для равномерного распределения студентов между лабораторными занятиями для защит, каждый студент заранее выбирает день своей защиты (таблицы для записи предоставляются преподавателем лабораторных занятий), количество защищающихся в один день студентов ограничено.

  • Студенту необходимо создать корректный PR за 24 часа до занятия, выбранного для защиты
  • Если день защиты, который выбрал студент, позднее дедлайна по созданию PR из табл. 2 (следующее после дедлайна занятие) - PR должен быть создан до дедлайна из табл. 2.
  • Т.е. PR, допускаемые до защиты, должны бы созданы до дедлайна из табл. 2, независимо от выбранного дня защиты, в ином случае - работа к защите не допускается (0 баллов).
  • Неявка на защиту в выбранный день без уважительной причины - 0 баллов за защиту без возможности пересдачи.

Крайний срок создания всех PR (с целью получения 2 баллов за них) и внесения в них изменений: 25.12.2022 23:59. Изменения любых PR после дедлайна не учитываются.

  • Исключение составляют PR не имеющие ни одного ревью от преподавателя, которые не были проверены менее чем за сутки до дедлайна (или позднее). В этом случае у студента есть возможность повторной проверки, если изменения внесены в течение 24-х часов после первой проверки.

Как проходит защита

Если вы выполнили все условия из предыдущего пункта, вы допускаетесь к защите лабораторной работы. Защита проходит очно. Вы получите задание, которое вы должны будете сделать без интернета (если вам прямо не укажут обратное). Задание состоит из двух частей: это теоретическая и практическая части. Вам будет дано на выполнение задания 45 минут, по истечении времени вы не сможете продолжить решать это задание.

Если вы выполнили задание, вы можете записаться в очередь на защиту и ждать приглашения преподавателя.

Оценка за защиту

За защиту максимум вы можете получить три балла. Каждый следующий пункт описывает, за что можно получить один балл:

  • Правильные ответы на вопросы по заданию, которое вы делали на защиту.
  • Правильные ответы на вопросы по коду лабораторной работы. Вы должны хорошо ориентироваться в коде и уметь его модифицировать.
  • Правильные ответы на вопросы по курсу. Темы вопросов к каждой лабораторной работе будут указаны на вики-странице курса. 

Регламент проведения экзамена

Если оценка согласно рейтингу не устраивает студента, или его результат по итогу рейтинга неудовлетворительно - студенту необходимо сдавать экзамен.

Основные положения по сдаче экзамена:

  • Полный отказ от результатов рейтинга (без возможности отката)
  • Экзамен состоит из:
  • Экзаменационный билет, содержащий две задачи на программирование
  • Реализация структуры данных
  • Реализация алгоритма(ов)
  • Устная беседа с преподавателем (теоретические вопросы по темам курса)
  • Для успешной сдачи экзамена (оценка > 2):
  • необходимо выполнение обеих частей билета в той или иной степени, т.е., например, если студент выполнил часть/все задачи, но не способен ответить на вопросы по теории = оценка  неудовлетворительно
  • при решении билета (программирование / теория) студенту запрещается пользоваться какими-либо материалами и программными средствами кроме IDE.

Пример экзаменационного билета

  1. Реализуйте бинарное дерево поиска, позволяющее хранить матрицы 2х2. Предложите и реализуйте три варианта весов узла подобного дерева.
  2. Реализуйте программу, осуществляющую центрированный обход бинарного дерева из первой задачи и сортирующую полученный массив матриц с помощью собственной сортировки слиянием.

Формирование оценки за экзамен

Баллы, заработанные в течение семестра сохраняются.

На ДК / допсессии защита лабораторных работ не осуществляется, студент может добрать баллы с помощью 3-х шагов:

  1. пройти или допройти онлайн-курс заранее до пересдачи, в нужный день баллы  будут засчитаны по следующему правилу: 1 выполненный элемент курса, содержащий задачу на программирование, дает 1.5 балла рейтинга.
  2. выполнить недостающие лабораторные работы и создать корректные PR с исходным кодом и отчетом не менее, чем за 3 дня до ДК/даты допсессии.
  3. решить дополнительные задачи, 1 задача = 2 балла рейтинга.

Шаги организованы с понижением приоритета: сначала баллы добираются за счет прохождения курса, затем, если баллов недостаточно - создаются PR по несданным лабораторным работам, и далее - выполняются доп. задачи.

Формирование оценки за курсовую работу

  • Оценка, полученная в результате защиты курсовой, аннулируется.
  • Студенты, защищающие курсовую работу повторно (неудачная первая попытка или повышение результата), защищаются по модифицированному регламенту: билет на защиту содержит две задачи на программирование, время выполнения билета увеличено до 60 минут.
  • Для студентов, защищающих курсовую работу впервые, сохраняется основной регламент проведения.
